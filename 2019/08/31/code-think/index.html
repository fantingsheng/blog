读《代码整洁之道》的一些零散想法

2019.08.31 Timfan 感悟  热度 ℃

最近写代码好像更多的时候在想怎么写出更简洁更高效的代码，之前看过一种说法，写代码的几个层次和阶段，先是能写，拼拼凑凑实现功能先，以函数写法为主，程序由一个个函数组成，函数间相互调用；其次是追求更好的写法，比如面向对象，同一类的问题可以更抽象，高效；再次是可以玩一些更高级的写法比如函数式，更加锻炼程序员的思维，完全颠覆正常的程序思维。

为什么要强调代码的整洁，里面提到了，其实我们大部分的时间都是在阅读代码，真正写代码的可能就是那一会儿。而且你写的代码可能是一个团队去维护，你得让你接手的人更容易上手去修改你的代码，反正代码的易读性你早晚有一天会意识到它的重要性。
