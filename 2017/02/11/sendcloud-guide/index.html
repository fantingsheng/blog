SendCloud邮件系统使用注意事项

2017.02.11 Timfan 技术  热度 ℃

SendCloud 邮件发送系统注意事项

1. 去掉代发的显示：保证发信域名和发信人地址一致，腾讯邮箱接收不再显示代发的方法：代码中将from参数写成xxx@your.domain.com your.domain.com 代表相应API_USER发信域名

2. 付费用户可以去掉邮件底部取消订阅按钮

3. 垃圾邮件防止策略，域名预热白皮书

1. 邮件进垃圾箱跟用户发信内容、发信域名评分、IP质量等情况息息相关，SendCloud密切关注着IP的备案情况和评分情况，以保证提供优质的IP供用户发信。当出现邮件进垃圾箱时，如果是触发邮件，SendCloud可以协助用户进行申诉；如果是批量邮件，SendCloud可以提供内容或者发信行为的调整意见

4. 无效邮件包括：

1. 收件人在sendcloud黑名单中
2. 收件人在取消订阅列表中
3. esp服务器不可达
4. 收件人格式错误
5. ip或者域名被拒
6. 收件人不存在
7. 发件人或者收件人被拒
8. 被判为垃圾邮件

5. 信誉度

用户在 SendCloud 平台的信誉评分. 「信誉度」和「当天请求额度」是高度正相关的两个数值. 当信誉度小于 0 时, 当天请求额度为 0 .
此评分是由 SendCloud 根据用户的邮件发送数据实时计算出来的
    * ++ 优质的「送达率」,「打开率」,「点击率」会促成加分
    * -- 大量的「无效地址」,「垃圾举报」,「取消订阅」则会促成减分

6. 各项统计指标说明：

1. 打开：用户打开邮件的次数，即使是同一封邮件，打开几次就计算几次。
2. 点击：用户点击邮件中链接的次数，即使是同一封邮件，点击几次就计算几次。
3. 独立打开：终端用户独立打开邮件的次数，同一封邮件多次打开算作一次。
4. 独立点击：终端用户独立点击邮件中链接的次数，同一封邮件多次点击算作一次。
5. 软退信（soft bounce）是指下列任何情况之一造成的邮件无法送达：
    1.  由于收件方邮件服务器的原因没有形成反应。这种情形可能是暂时的，也可能会持续一段时间，这期间将形成退信；
    2.  用户电子邮箱空间没有足够容量接收邮件。邮箱空间已满的情况下会形成退信，如果用户对电子邮箱进行清理，删除部分邮件之后，这个邮箱还有可能正常使用。
6. 垃圾邮件举报：用户点击取消订阅选择垃圾邮件选项
